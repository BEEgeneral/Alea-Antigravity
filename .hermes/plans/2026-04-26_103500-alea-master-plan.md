## 2. Arquitectura Actual vs. Objetivo

### Estado Actual (Parcialmente Roto)

```
[Frontend] 
    ↓ HTTP (SSE)
[route.ts] ──→ [HermesTools.getDefaultTools()] → MiniMax API
    ↓ (tool_calls detectados)
[executeToolCall() en route.ts] → ❌ NO EXISTE
    ↓
[HermesToolExecutor] (existe pero no se llama desde route)
    ↓
[InsForge DB]
```

### Estado Objetivo (Funcional)

```
[Frontend]
    ↓ SSE
[route.ts] 
    ├── getMemoryContext() → InsForge DB (corregido)
    ├── HermesTools.getTools()
    ├── MiniMax API (streaming)
    ├── executeToolCall() → ToolExecutor (corregido)
    │   ├── HermesToolExecutor (todas las tools)
    │   └── InsForge DB
    └── Streaming response con tool_results re-enviados al modelo
```

#### F3.3: Streaming Completo
El flujo actual de streaming es:
1. Enviar mensaje → MiniMax streaming
2. Si hay tool_calls → enviar tool_calls al cliente
3. ❌ NO hace segunda llamada al modelo

Fix completo:
```typescript
// Después de recibir todos los tool_calls y ejecutarlos:
// 1. Construir mensajes con tool_results
messages.push({ role: 'assistant', content: fullContent, tool_calls: toolCallsBuffer });
for (const tr of toolResults) {
  messages.push({
    role: 'tool',
    tool_call_id: tr.toolCallId,
    content: typeof tr.result === 'string' ? tr.result : JSON.stringify(tr.result),
  });
}
// 2. Hacer segunda llamada a MiniMax SIN streaming
const finalResponse = await openai.chat.completions.create({
  model: MINIMAX_MODEL,
  messages,
  max_tokens: 4096,
  temperature: 0.7,
});
// 3. Enviar respuesta final al cliente
```

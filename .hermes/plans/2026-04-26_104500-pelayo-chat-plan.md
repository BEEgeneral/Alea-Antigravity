# Plan: Chat Pelayo Limpio en /praetorium

**Fecha:** 2026-04-26  
**Objetivo:** Reemplazar `/praetorium` con una página limpia que solo tenga Chat Pelayo

---

## Lo que existe ahora

- `/praetorium/page.tsx` — 2985 líneas, dashboard admin con TODO (pipeline, CRM, properties, agents, chat widget, agenda, etc.)
- `/centrion/pelayo/page.tsx` — Página Pelayo standalone (voice + notificaciones) → USUARIO NO LO QUIERE
- `src/components/admin/PelayoChat.tsx` — Componente widget flotante de chat (BUENO, reutilizable)
- `src/app/api/hermes/chat/route.ts` — API endpoint (funciona con auth via cookie)
- Auth: cookie `insforge_token`, verificación via `/api/auth/me`

---

## Lo que hay que hacer

### PASO 1: Reemplazar `/praetorium/page.tsx`

Nueva página mínima que:
1. Verifica auth (cookie `insforge_token`)
2. Si no está auth → redirect a `/login`
3. Si está auth → muestra `PelayoChat` en pantalla completa (no flotante)

El componente `PelayoChat` ya existe y es bueno — tiene:
- Streaming SSE
- Voice input (Web Speech API)
- TTS (MiniMax TTS API)
- Tool results
- Mensajes guardados
- UI profesional

**Solo hay que:**
1. Quitar el `isOpen`/`onClose` wrapper (no es floating widget, es página completa)
2. Conectar auth correctamente (actualmente PelayoChat no pasa token al API)

### PASO 2: Fix Auth en PelayoChat

**Problema actual:** `PelayoChat.tsx` llama `/api/hermes/chat` pero NO pasa el token Bearer.
```typescript
// Así está ahora (MAL):
const response = await fetch('/api/hermes/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ message: messageText, history, stream: true }),
});

// El endpoint espera: Authorization: Bearer <token>
// Pero PelayoChat no lo envía
```

**Fix:** Obtener token de `localStorage.getItem('insforge_token')` y enviarlo.

### PASO 3: Eliminar Centurion

- `/src/app/centrion/` (si existe) → eliminar o dejar pero sin acceso
- El usuario dice que no lo quiere

### PASO 4: Verificar API route de Herme

El endpoint `/api/hermes/chat` espera:
- Cookie `insforge_token` O
- Header `Authorization: Bearer <token>`

Verificar que la auth funciona correctamente.
